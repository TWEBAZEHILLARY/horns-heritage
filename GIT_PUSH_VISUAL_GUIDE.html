📤 Git Push Step-by-Step

Complete guide to merge with GitHub and enable CI/CD

Progress

1. Clone
2. Copy Files
3. Git Add
4. Commit
5. Push
6. Test

Choose Your Operating System

Using Terminal/Command Line

1
Clone Your Repository
Copy your GitHub repo to your computer
$ Run this command: git clone https://github.com/TWEBAZEHILLARY/horns-heritage.git
Then navigate into the folder:
$ Run this: cd horns-heritage
2
Copy Files from Workspace
Copy these files/folders from your workspace into the cloned repository:
  • wrangler.toml
  • package.json
  • .gitignore
  • src/ (entire folder)
  • .github/ (entire folder)
  • public/ (entire folder)
  • assets/ (entire folder)
  • _ds/ (entire folder)
How to copy: Open File Manager/Finder and drag files from workspace into the horns-heritage folder you just created.
3
Stage Files with Git
Tell Git to track all your changes:
$ Run this: git add .
The dot (.) means "add all files"
4
Commit Changes
Save your changes with a message:
$ Run this: git commit -m "Merge: Add deployment configuration and CI/CD pipeline"
The -m flag adds a commit message
5
Push to GitHub
Upload your changes to GitHub:
$ Run this: git push origin master
Note: If you get an error saying "master doesn't exist", try: git push origin main
6
Verify on GitHub
Check that your files are now on GitHub:
  1. Go to: https://github.com/TWEBAZEHILLARY/horns-heritage
  2. You should see wrangler.toml, package.json, src/, public/, .github/ at the root
  3. If yes, ✅ push was successful!
✅ After pushing successfully, continue with:
1. Add GitHub secrets (CLOUDFLARE_API_TOKEN and CLOUDFLARE_ACCOUNT_ID)
2. Test by making a small change and pushing again

Using GitHub Desktop (Easier!)

1
Open GitHub Desktop
Download from: https://desktop.github.com/
2
Clone Your Repo
In GitHub Desktop:
  1. Click "File" → "Clone Repository"
  2. Select "TWEBAZEHILLARY/horns-heritage"
  3. Choose where to save it
  4. Click "Clone"
3
Copy Files
Manually copy files from workspace to the cloned folder:
  • wrangler.toml
  • package.json
  • .gitignore
  • src/ folder
  • .github/ folder
  • public/ folder
4
Commit in GitHub Desktop
  1. GitHub Desktop will detect changes automatically
  2. In the "Changes" tab, you'll see all modified files
  3. Enter commit message: "Merge: Add deployment configuration"
  4. Click "Commit to master"
5
Push to GitHub
  1. Click "Push origin" button (top right)
  2. Wait for it to complete
  3. Done! Files are on GitHub
✅ GitHub Desktop makes it easy!
Much simpler than command line - visual and intuitive.

Using GitHub Web Interface

1
Go to Your GitHub Repo
Navigate to: https://github.com/TWEBAZEHILLARY/horns-heritage
2
Upload Files
  1. Click "Add file" → "Upload files"
  2. Drag and drop these files:
  • wrangler.toml
  • package.json
  • .gitignore
  1. Click "Commit changes"
3
Create Folders
  1. Click "Add file" → "Create new file"
  2. Type path: src/index.js
  3. Paste the content from your workspace
  4. Click "Commit new file"
  5. Repeat for .github/workflows/deploy.yml
4
Upload Folders
Repeat the "Upload files" process for:
  • public/ (all files)
  • assets/ (all images)
  • _ds/ (all design system files)
⚠️ Note: Web interface is slower for many files. GitHub Desktop or Terminal is faster for bulk uploads.

After Pushing to GitHub

Step 1: Add GitHub Secrets
Go to: https://github.com/TWEBAZEHILLARY/horns-heritage/settings/secrets/actions
  1. Click "New repository secret"
  2. Add CLOUDFLARE_API_TOKEN (your Cloudflare API token)
  3. Click "Add secret"
  4. Click "New repository secret" again
  5. Add CLOUDFLARE_ACCOUNT_ID (your Cloudflare Account ID)
  6. Click "Add secret"
Step 2: Test Deployment
  1. Make a small change to a file (e.g., edit index.html)
  2. Commit and push the change
  3. Go to: https://github.com/TWEBAZEHILLARY/horns-heritage/actions
  4. Watch "Deploy to Cloudflare Workers" workflow run
  5. Once ✅ green, visit: https://horns-heritage.workers.dev
  6. Your site is LIVE! 🎉
