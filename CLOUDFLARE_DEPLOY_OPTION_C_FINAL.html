⚠️Why Bundling Didn't Work

Your site contains:

  • 9 hero cow images (JPGs)
  • 4 video files (MP4s) — cattle-video.mp4, cow-2.mp4, cow-3.mp4, cow-pasture.mp4
  • 20+ product images
  • CSS files, JS scripts, design system assets

Total: 30+ MB — too large to bundle into a single HTML file.

Cloudflare's limit: 30 MB max file upload.

✅The Solution

Don't bundle. Deploy your entire project folder as-is.

Cloudflare Pages will serve all your files from the root level. Your relative paths will work perfectly — no broken links.

Why This Works

When you deploy a folder to Cloudflare Pages, it becomes the web root (/):

  • Your index.html is served at /
  • Your hh-enhance.css is served at /hh-enhance.css
  • Your assets/ankole-1.jpg is served at /assets/ankole-1.jpg
  • Your assets/cattle-video.mp4 is served at /assets/cattle-video.mp4

All relative paths in your HTML work exactly as written. No 404s, no broken images, no missing videos.

2Deploy to Cloudflare (Dashboard Method)

Easiest, no terminal required.

  1. Open https://dash.cloudflare.com/
  2. Log in with your Cloudflare account
  3. Select your domain (or create a new project)
  4. Go to Pages section
  5. Click "Create project"
  6. Choose "Upload files" (NOT "Connect to Git")
  7. Drag and drop your entire project folder
    • Or: Click to select, then choose all files in your project root
    • Include: index.html, all .css, all .js, assets/, _ds/
  8. Build settings:
    Field Value
    Framework None
    Build command (Leave empty)
    Build output directory . (current directory)
  9. Click "Save and Deploy"
✅ Live! Cloudflare assigns you a URL: horns-heritage-abc123.pages.dev

To use your own domain:

  1. In Cloudflare Dashboard, go to your domain
  2. Click Pages → your project
  3. Go to Custom domains
  4. Add your domain (e.g., horns-heritage.com)

2BDeploy to Cloudflare (Terminal Method)

If you prefer command line:

npm install -g wrangler
wrangler login
wrangler pages deploy . --project-name=horns-heritage

That's it! Your site is deployed.

3Verify Everything Works

After deployment, check that all assets load correctly.

  1. Open your live site (e.g., horns-heritage.pages.dev)
  2. Press F12 to open DevTools
  3. Click the "Network" tab
  4. Reload the page (Cmd+R or Ctrl+R)
  5. Scroll through the list and look for RED items:
    • ✅ Green/200 = File loaded successfully
    • ❌ Red/404 = File not found (broken path)
File Expected If Missing (404)
index.html 200 Main page — always required
hh-enhance.css 200 Check file is in root
app.js 200 Check file is in root
assets/ankole-1.jpg 200 Check assets/ folder uploaded
assets/cattle-video.mp4 200 Check assets/ folder uploaded
_ds/... 200 Check _ds/ folder uploaded
✅ All 200s? Your site is fully live! All assets are loading.
❌ Any 404s? A file didn't get uploaded. Go back and ensure all folders (assets/, _ds/, etc.) are included in your Cloudflare upload.

4Track Your Edits (How They Link)

Document what you changed and preserve the history.

Option A: Git + GitHub (Recommended)

Push your project to GitHub, then connect to Cloudflare for auto-deploys.

git add index.html hh-enhance.css assets/

Benefits:

  • GitHub shows diff (exactly what changed)
  • Cloudflare auto-deploys on push (connect via Pages → Connect to Git)
