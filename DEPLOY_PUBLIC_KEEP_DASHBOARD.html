🎯 Deploy Public Site + Keep Dashboard

Horns & Heritage — Deploy Public to Cloudflare, Keep Dashboard on Current Server

1Understand Your Two Apps

🌐 PUBLIC WEBSITE

  • What visitors see
  • index.html (main page)
  • All CSS/JS files
  • assets/ (images, videos)
  • Reads cattle data
  • Shows news feed

Deploy to: Cloudflare Pages

URL: horns-heritage.pages.dev

📋 SALES DASHBOARD

  • What you manage
  • Sales Dashboard.html
  • admin-news-dashboard.html
  • Add/edit cattle
  • Manage news posts
  • Writes data to HHApi

Deploy to: Current server (keep it)

URL: localhost:5000 (or your server)

✅ Both apps sync via HHApi: When you add cattle in the Dashboard, the Public Website automatically shows it. They read/write the same data store.

2Prepare Files for Cloudflare (Public Only)

Create a new folder for Cloudflare deployment:

mkdir cloudflare-deploy
cd cloudflare-deploy

# Copy ONLY public site files (NOT dashboard files)
cp ../index.html .
cp ../hh-*.css .
cp ../hh-*.js .
cp ../app.js .
cp ../tilt-3d.* .
cp -r ../assets .
cp -r ../_ds .

# Do NOT copy:
# - Sales Dashboard.html
# - admin-news-dashboard.html
# - dashboard-*.* files
# - public/ folder
✅ Result: A folder with ONLY public site files, ready for Cloudflare.

3Deploy Public Website to Cloudflare

Go to https://dash.cloudflare.com/

  1. Pages → Create project → Upload files
  2. Drag your cloudflare-deploy folder
  3. Build settings:
    • Framework: None
    • Build command: (leave empty)
    • Build output: .
  4. Deploy
✅ Live at: horns-heritage.pages.dev (Cloudflare URL) or your custom domain

4Keep Dashboard Running on Your Server

Your Sales Dashboard stays where it is.

Option A: Local Development Server

cd /path/to/your/project
python -m http.server 5000
# or: npx http-server -p 5000

Access dashboard at: http://localhost:5000/Sales%20Dashboard.html

Option B: Production Server (AWS, Heroku, etc.)

Deploy the public/ folder to your server. The dashboard stays accessible.

Files that stay on your server:

  • public/Sales Dashboard.html
  • public/admin-news-dashboard.html
  • public/dashboard-*.* files
  • public/hh-api.js (for data syncing)
⚠️ Important: Keep hh-api.js on your server so the dashboard can write data to HHApi (localStorage or Workers/KV).

5How Data Syncs Between Both Sites

HHApi is your data bridge.

Event Dashboard (Your Server) Public Site (Cloudflare)
You add cattle Dashboard writes to HHApi (localStorage or Workers/KV) Public site reads from HHApi, displays new cattle
You post news Dashboard writes to HHApi Public site reads from HHApi, shows new post
Customer browses — Public site displays cattle + news from HHApi
Customer buys — Public site calls HHApi to mark cattle as sold
✅ Both apps stay in sync because they share the same HHApi data store.

6Access Both Sites After Deployment

Site URL Purpose
Public Website horns-heritage.pages.dev What customers see
Sales Dashboard http://localhost:5000/Sales%20Dashboard.html What you manage
News Dashboard http://localhost:5000/admin-news-dashboard.html Manage news posts

Both run simultaneously. They don't interfere with each other.

7Verify Everything Works

  1. Open public site: horns-heritage.pages.dev
    • Press F12 → Network
    • Verify all files load (200 status)
    • Check images, videos, CSS display correctly
  2. Open dashboard: http://localhost:5000/Sales%20Dashboard.html
    • Verify dashboard loads
    • Add a test cattle item
  3. Refresh public site — should show your new cattle item
    • Data synced via HHApi ✅
✅ Complete! Public site on Cloudflare, Dashboard on your server, both synced.
