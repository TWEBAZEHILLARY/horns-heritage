Deploy fix

Why the deploy failed, and how to fix it

Cloudflare's build machine clones your GitHub repo into /opt/buildhome/repo. Your repo has no public/ folder — every site file sits at the repo root, and the assets/ images folder was never pushed. But wrangler.toml says directory = "./public", so Wrangler stops with "the directory does not exist".

The fix: a tiny build script, build-public.mjs, now assembles ./public from your root files on every deploy — and Wrangler is updated to v4, which clears the version warning. Three files changed: build-public.mjs (new), package.json, .github/workflows/deploy.yml.

Step 1 — Commit and push the changed files

1

From your local repo folder, copy in the three files from this project, then:

git add build-public.mjs package.json .github/workflows/deploy.yml assets
git commit -m "fix: build ./public before deploy, wrangler v4"
git push origin master

Note assets in the git add — your images folder is not in the repo yet. Without it the site deploys but every image 404s.

Step 2 — Update the Cloudflare build settings

2

In the Cloudflare dashboard: Workers & Pages → horns-heritage → Settings → Build, set:

SettingValue
Build commandnpm install && npm run build
Deploy commandnpx wrangler deploy

This matches the Wrangler docs you linked — Wrangler installed as a dev dependency and always run through npx wrangler.

Step 3 — The next error you will hit (fix it now)

3

Your wrangler.toml still has placeholder KV ids (REPLACE_WITH_CATTLE_KV_ID). Once the public-folder error is gone, deploy will fail on these. Create the namespaces once and paste the printed ids into all four id fields in wrangler.toml:

npx wrangler kv namespace create CATTLE_KV
npx wrangler kv namespace create NEWS_KV

Verify locally before pushing

npm install
npm run build        # should print: ✓ built ./public
npx wrangler deploy --dry-run
What gets deployed

The script copies index.html, Sales Dashboard.html, admin-news-dashboard.html, all CSS/JS, and assets/ into public/. It deliberately skips worker.js, the setup guides, and all .md docs so they are never publicly served. public/ stays out of git — it is rebuilt on every deploy.
