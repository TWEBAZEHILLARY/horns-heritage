🚀 Cloudflare Pages Deployment

Horns & Heritage — Complete Step-by-Step Guide

1Check Your File Structure

Before deploying, verify your files are organized correctly for Cloudflare.

Cloudflare Pages expects ONE root folder with all assets at the top level.

📁 your-project/
├── 📄 index.html
├── 📄 hh-enhance.css
├── 📄 hh-redesign.css
├── 📄 hh-discover.css
├── 📄 app.js
├── 📄 tilt-3d.js
├── 📁 assets/
│   ├── 📷 hero-image.jpg
│   ├── 📷 logo.png
│   └── ...
├── 📁 _ds/
│   └── urbanloft-design-system-6db9302f-8fa0-4943-bea9-04c287185da1/
│      ├── _ds_bundle.js
│      ├── styles.css
│      └── tokens/
⚠️ Important: If your files are nested in src/, public/, or other folders, you must move them to the root BEFORE deploying.

2Choose Your Deployment Method

Pick the approach that works best for your workflow:

🎯 Option A: Simple Dashboard Upload (Easiest)

No Git, no terminal. Upload files directly from Cloudflare's web interface.

  • Best for: Quick deployments, testing, one-off uploads
  • Time: 2-3 minutes
  • Edits: Upload updated files each time (not automatic)

🔄 Option B: GitHub + Auto-Deploy (Recommended)

Push to GitHub → Cloudflare auto-deploys. Professional workflow.

  • Best for: Team projects, version control, automatic updates
  • Time: 5 minutes setup, then instant deploys
  • Edits: Automatically tracked in Git history

📦 Option C: Bundle to Single HTML (Safest)

All code + assets in ONE file. No broken relative paths.

  • Best for: Maximum reliability, offline-ready
  • Time: 2 minutes
  • Edits: Single file to edit and deploy

3ADeploy via Cloudflare Dashboard (Option A)

If you chose Option A: Simple Dashboard Upload

  1. Go to https://dash.cloudflare.com/
  2. Log in with your Cloudflare account
  3. Select your domain (or create a new project)
  4. Go to Pages → Click Create project
  5. Choose "Upload files" (NOT "Connect to Git")
  6. Drag and drop your entire project folder (or select files)
  7. Set build settings:
    Field Value
    Framework None
    Build command (Leave empty)
    Build output directory . (current directory)
  8. Click "Deploy site"
✅ Your site is live! Cloudflare assigns you a URL like horns-heritage-abc123.pages.dev

To verify assets loaded:

  1. Open your Cloudflare site in a browser
  2. Press F12 to open DevTools
  3. Go to the Network tab
  4. Reload the page (Cmd+R or Ctrl+R)
  5. Check the list — all files should show 200 (success) not 404 (missing)

3BDeploy via GitHub + Auto-Deploy (Option B)

If you chose Option B: GitHub + Auto-Deploy

Part 1: Push Your Code to GitHub

cd /path/to/your/horns-heritage
git init
git add .
git commit -m "Initial commit: Horns & Heritage site"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/horns-heritage.git
git push -u origin main
📝 Note: Replace YOUR-USERNAME with your actual GitHub username.

Part 2: Connect GitHub to Cloudflare

  1. Go to https://dash.cloudflare.com/
  2. Select your domain → Pages
  3. Click "Create project" → "Connect to Git"
  4. Authorize GitHub (sign in if prompted)
  5. Select your repository (horns-heritage)
  6. Build settings:
    Field Value
    Framework None
    Build command (Leave empty)
    Build output directory .
  7. Click "Save and Deploy"
✅ Now every time you push to GitHub, Cloudflare auto-deploys!

Future updates are simple:

git add index.html hh-enhance.css ...
git commit -m "Update hero section styling"
git push

3CBundle to Single HTML (Option C)

If you chose Option C: Bundle to Single HTML

This approach bundles ALL CSS, JS, images, and fonts into ONE HTML file. No more broken relative paths.

Using the Design Tool

  1. In your design editor, use: super_inline_html()
  2. Input: index.html
  3. Output: horns-heritage-standalone.html

This creates a single file with everything embedded.

Then Deploy to Cloudflare

  1. Go to https://dash.cloudflare.com/
  2. Pages → Create project → Upload files
  3. Upload only horns-heritage-standalone.html
  4. Deploy
✅ Guaranteed to work! No file paths can break because everything is in one file.

4Track Your Edits (Version Control)

Document which files you changed and why.

Option 1: Git History (Automatic)

If you used Option B (GitHub), your edits are automatically tracked:

git log --oneline
# Shows all commits with messages like:
# abc1234 Update hero section styling
# def5678 Fix cart calculation
# 123abcd Initial commit

Option 2: Create an EDIT_HISTORY.md File

Document your changes manually:

# Edits to Horns & Heritage

## 2026-06-13
- **index.html**: Updated hero background to #3C6E54
  - Line 456: Changed --primary-green color
- **hh-enhance.css**: Added card hover effect (line 234-240)

## 2026-06-12
- **app.js**: Fixed cart calculation bug (line 145)

Commit this file to Git:

git add EDIT_HISTORY.md
git commit -m "Add edit history documentation"
git push

Option 3: Add Comments in Code

Mark edited sections with comments:

/* EDITED: 2026-06-13 - Custom theme colors */
:root {
  --primary-green: #3C6E54; /* CUSTOM: was #5A8970 */
  --secondary-green: #5C7B66;
}

5Verify Everything Works

After deployment, confirm all assets load correctly.

In Your Browser

  1. Open your live site (e.g., horns-heritage.pages.dev or your custom domain)
  2. Press F12 to open DevTools
  3. Click the "Network" tab
  4. Reload the page (Cmd+R or Ctrl+R)
  5. Look for red/404 errors:
    • ✅ All files show 200 = Success
    • ❌ Any files show 404 = Missing file
File Expected Status If 404: Fix
index.html 200 Main file — should always load
hh-enhance.css 200 Check file is in root folder
app.js 200 Check file is in root folder
assets/image.jpg 200 Check assets/ folder exists
_ds/... 200 Check _ds/ folder exists
⚠️ If you see 404s: Your file structure is wrong. Go back to Step 1 and ensure all files are in the root folder, or use Option C (Bundle HTML) to avoid path issues.

6Future Deployments

If You Used Option A (Dashboard Upload)

Each time you make changes:

  1. Go to https://dash.cloudflare.com/ → Pages
  2. Click your project
  3. Click "Re-deploy from GitHub" (or upload files again)

If You Used Option B (GitHub Auto-Deploy)

Just push to GitHub — Cloudflare deploys automatically:

git add .
git commit -m "Your change description"
git push origin main

If You Used Option C (Standalone HTML)

Re-bundle and re-upload:

  1. Recreate the bundled HTML file
  2. Upload to Cloudflare Pages
